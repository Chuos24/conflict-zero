**Severidad:** Alta

Se encontró que `app/main.py` importaba varios routers en el `__init__.py` pero NO los incluía en la aplicación FastAPI. Esto significaba que las siguientes funcionalidades estaban implementadas en código pero INACCESIBLES vía API:

| Router | Estado anterior | Estado actual |
|--------|----------------|---------------|
| admin_router | Importado pero NO incluido | ✅ Incluido |
| notifications_router | Importado pero NO incluido | ✅ Incluido |
| invitations_router | Ni importado ni incluido | ✅ Incluido |
| certificates_router | Ni importado ni incluido | ✅ Incluido |
| features_router | Ni importado ni incluido | ✅ Incluido |
| tags_router | Ni importado ni incluido | ✅ Incluido |
| templates_router | Ni importado ni incluido | ✅ Incluido |
| payments_v2_router | En __init__.py pero NO en main.py | ✅ Incluido |
| payments_admin_router | Archivo existente, NO exportado | ✅ Exportado e incluido |

**Impacto:** Antes de la corrección, los usuarios no podían acceder a: admin, notificaciones, invitaciones, certificados, features, tags, templates, pagos v2 ni pagos admin.

### 2. ERROR EN PAYMENTS_ADMIN.PY
**Severidad:** Media

Línea 205: `u.status` → El modelo `User` NO tiene columna `status`. Tiene `is_active` (boolean).
- Corregido a `u.is_active`

### 3. FRONTEND: BUILD ROTO POR CÓDIGO LEGACY MVP
**Severidad:** Alta

El directorio `frontend/src/` contenía código del MVP experimental (React puro con Vite) que usaba:
- `react-router-dom` (no instalado)
- `zustand` (no instalado)

Esto causaba que `npm run build` fallara completamente.

**Solución:** Renombrado `src/` → `src_backup_mvp/`. La app actual usa Next.js 14 App Router (`frontend/app/`).

### 4. POSTCSS.CONFIG.JS EN FORMATO ESM
**Severidad:** Media

Next.js 14 espera CommonJS en `postcss.config.js`. El archivo usaba `export default`.
- Convertido a `module.exports`

### 5. TAILWIND.CONFIG.JS EN FORMATO ESM + PATHS INCORRECTOS
**Severidad:** Media

- Usaba `export default` en lugar de `module.exports`
- Los `content` paths apuntaban a `./index.html` y `./src/**/*` (legacy)
- Actualizados a `./app/**/*`, `./components/**/*`, etc.
- Agregada configuración de colores CSS variables para compatibilidad con shadcn/ui (`border-border`, etc.)

### 6. NAV DUPLICADA EN DASHBOARD LAYOUT
**Severidad:** Baja

"Mi Red" aparecía dos veces en la barra de navegación del dashboard.
- Eliminada entrada duplicada.
